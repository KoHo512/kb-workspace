사용자 -> 웹 브라우저 -> 서버 : 자원(URL, 도메인)을 요청
서버가 보내준 페이지를 웹 브라우저가 화면에 렌더링

=> 각 URL 요청은 서버에 새로운 페이지 로그 유발
=> 서버 부하가 심했음

=> 개선하기 위해 나온 게 SPA


SPA(Single Page Application) : Vue.js, React, ...
- 첫 페이지 로딩 시, 서버로부터 필요한 모든 자원을 한 번에 불러옴
- 사용자와 상호작용을 통해 동적으로 화면 갱신
    (웹 브라우저에서 URL이 변경되더라도 서버에게 새로운 페이지 로드를 요청하지 않음
    JavaScript를 통해 화면에 표시될 콘텐츠를 동적으로 렌더링)


Vue.js에서 제공하는 vue-router란?
- Vue 어플리케이션에서 브라우저의 URL과 애플리케이션의 view를 연결
- SPA에서 URL의 변화에 따라 적절한 컴포넌트를 화면에 렌더링하도록 지원하는 기능

- vue-router 기본 원리

- 라우팅 기법
    1. 동적 라우팅 매칭
    2. 중첩 라우팅
    3. 내비게이션
    4. 라우터 가드


1. 라우팅 컴포넌트 생성 (화면) - views/~
2. 라우터 인스턴스 생성하고 등록 - router/index.js
3. 라우터 인스턴스 사용 등록 - main.js
4. 화면 등록 - app.vue
