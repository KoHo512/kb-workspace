Vue.js 프레임워크 : 웹 어플리케이션의 복잡한 코드, 구조를 컴포넌트로 해결
웹 어플리케이션을 유연하고 모듈화된 구조로 개발 - 관리 용이

1. 컴포넌트의 슬롯, 데이터 공유 기능
    - 컴포넌트 : 자바스크립트 기반 프레임워크에서 어플리케이션을 구성하과 관리하는 방법
    
    - 과거에는 페이지 단위로 개발을 했으나 비효율적
    => 더 세분화환 단위 필요 : CBA(Component Based Architecture)
        - 한 웹 App을 여러 컴포넌트로 구분해 컴포넌트 단위로 설계하는 방식
        - 컴포넌트 : 특정 기능이나 작업을 독립적으로 수행하기 위한 코드 단위

    - 재사용성 : 같은 기능이나 작업이 필요한 곳에서 재사용 가능
    - 독립성 : 컴포넌트는 독립적으로 작동해야 함 - 다른 컴포넌트에 영향 주지 않고 수정/교체 가능
    - 모듈성 : CBA는 웹 어플리케이션을 개별 단위로 구분하기 때문에 복잡성을 줄일 수 있음
    - 확장성 : 컴포넌트에서 구성 요소를 추가하거나 수정해서 새로운 동작이나 기능 제공 가능
    - 캡슐화 : 컴포넌트 내부의 복잡성을 숨기고 단순하게 기능만 사용할 수 있는 인터페이스 제공
