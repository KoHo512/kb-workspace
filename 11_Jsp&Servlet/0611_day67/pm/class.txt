HttpServlet
- GET/POST 요청에 맞도록 doGet(), doPost() 방식 제공
- HttpServlet을 상속 받은 클래스 객체는 톰캣이 자동으로 객체로 만들어 관리 - 개발자는 신경쓰지 않아도 ok
- 멀티 스레드에 의해 동시에 실행될 수 있도록 처리되어 있음

서블릿(Servlet)
- 요청을 처리해서 응답하는 것을 목적으로 설계
- 웹(WWW) 환경에서 개발자가 직접 객체를 생성/관리하는 대신 WAS에서 객체 관리 (서블릿 컨테이너, 엔진)

SampleServlet
- /sample 요청을 처음 넣었을 때 init() 수행 후 doGet() 호출
- 이후, 여러 번의 /sample 요청에도 불구하고 init()은 다시 실행 x - doGet()만 호출
- @this의 결과값이 모두 같음 => 동일하게 하나의 객체로 처리됨

HttpServletRequest
- Http 메세지 형태로 들어오는 요청에 대한 정보를 파악하기 위해 제공 (읽기 전용 기능 제공)
- HttpHeader : Http 헤더 내용 찾아내는 기능
- 사용자 관련 : 접속한 사용자의 정보
- 요청 관련 : GET/POST 정보, 사용자가 호출한 url
- 쿼리 스트링 : 데이터 추출
- 쿠키 관련 : 브라우저가 전송한 쿠키 정보
- 전달 관련
- 데이터 관련 : setAttribute(), getAttribute()

HttpServletResponse
- 쓰기 기능 담당
- MIME 타입 : setContentType("text/html)
