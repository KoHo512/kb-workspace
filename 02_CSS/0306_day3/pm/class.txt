1. CSS(Cascading Style Sheets)가 왜 필요한가?
    - HTML : 정보를 표현하는 역할 - 구조
    - CSS : HTML 문서를 시각적으로 꾸미는 역할
    - JavaScript : 정적인 HTML 문서에 동적인 행위를 부여하는 역할 - 동작
    => 가독성과 코드 관리 효율을 높이고 재사용성과 독립성 보장

2. CSS 적용
    - HTML의 style 태그 안에 작성
    - 외부 CSS 파일로 작성 후 HTML의 link 태그로 적용 

3. 외부 CSS 파일 적용

4. 글씨 크기, 정렬
    - Google 검색해서 MDN
    - 폰트 크기 지정 속성 : font-size
    - 정렬 속성 : text-align

5. 선택자(Selector)
    - HTML에 스타일을 적용할 때, HTML 요소를 지정하는 역할

    - 모든 태그에 스타일을 적용하려면 일일이 태그를 적어야 할까?
    - 첫 번째 p 태그에만 원하는 CSS를 적용할 수 없을까?
    
    => 이를 보완하기 위해 5가지 선택자
    
    - 전체 선택자(*)
    - 유형(태그) 선택자 : h1, p, img 등 특정 태그를 지정
    - 아이디 선택자(#) : 아이디로 요소(태그)를 식별 - 특정 아이디로 요소 하나를 선택
    - 클래스 선택자(.) : 특정 클래스의 요소들을 선택 - 여러 요소에 스타일 적용하고 싶을 때
    - 복합 선택자 : 부모-자식 태그 복합적으로 사용 - 서로의 관계와 위치를 유용하게 결합
        - 자손 선택자
        - 자식 선택자
        - 인접 형제 선택자
        - 일반 형제 선택자

CSS 배치 속성 - 반응형 디자인

1. 배치 속성
    - display : block, inline

2. 박스 모델(Box Model) : margin, border, padding

    - block과 inline이라는 개념은 왜 존재할까?
    => 보기 좋게 배치하기 위해

    - 요소를 원하는 위치에 배치하려면 태그마다 특징을 알아야 함
    
        - block
        - inline
        - box model

    - 박스 모델 : 모든 HTML 요소를 감싸는 상자
        - 구성 : 요소(element), 패딩(padding), 테두리(border), 마진(margin)

3. 반응형 웹 페이지 : 다양한 디바이스(화면 크기)에 맞춰 자동으로 요소를 배치하는 웹페이지
