Spring이 객체 지향 구조를 설계할 때 받아들인 개념 : 의존성 주입
- 객체와 객체 간의 관계를 어떻게 더 유연하게 유지할 것인가
- 객체의 생성과 관계를 어떻게 효과적으로 분리할 것인가

모든 Controller는 Service 객체를 이용해야 db 사용 가능
= Controller는 Service 객체에 의존적

의존성(Dependency) : 하나의 객체가 자신이 해야할 일을 수행하기 위해 다른 객체의 도움이 필수적인 관계

의존성을 해결하기 위해
컨트롤러에서 서비스 객체를 직접 생성하거나 하나의 객체만을 생성해 활용

스프링 프레임워크는 프레임워크 자체에서 지원
- 다양한 방식으로 필요한 객체 찾아 사용할 수 있게 지원
- XML 설정, 자바 설정(어노테이션)


스프링 프레임워크
- 자체적으로 객체를 생성하고 관리하면서 필요한 곳에 객체를 주입(inject)하는 역할
- 이 역할을 수행하기 위해 설정 파일(xml)이나 어노테이션을 이용해 환경 세팅을 해야 함


초기 스프링에서는 @Autowired를 멤버 변수에 할당하거나 Setter를 작성하는 방식을 많이 사용
스프링 3 이후, 생성자 주입 방식을 더 많이 활용 & 권장

생성자 주입 방식
규칙 1. 주입 받아야 하는 객체의 변수는 final 작성
규칙 2. 생성자를 이용해 해당 변수를 생성자의 파라미터로 지정


스프링의 빈(Bean)으로 지정되는 객체들
- 모든 클래스의 객체가 스프링의 빈으로 처리되는 것은 아님
- 스프링 빈으로 등록되는 객체들은 "핵심 배역" 역할을 맡은 객체
    - DTO, VO : "데이터"를 중심으로 두고 설계된 객체 => 빈으로 등록되지 X
