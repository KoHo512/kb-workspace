PRG 패턴
- POST 방식의 요청에 대한 처리를 하고 Redirect 해서 GET 방식으로 특정 페이지로 이동하는 패턴
- 스프링 MVC에서는 RedirectAttributes 타입 제공 => Model
    - addAttribute(키, 값); - 리다이렉트할 때까지 쿼리 스트링 값 저장
    - addFlashAttribute(키, 값); - 일회용 데이터 전달 (사용되면 값 삭제)

스프링MVC 예외 처리
- 컨트롤러에서 발생하는 예외를 처리하는 가장 일반적인 방법 : @ControllerAdvice
- 선언된 클래스는 빈으로 등록해 처리
- NumberFormatException 예외 발생했을 때 처리하는 메소드 @ExceptionHandler(NumberFormatException).class) 등록
- @ResponseBody : 문자열이나 Json 객체를 브라우저에게 그대로 전송하는 응답 객체


HTTP Status
400번대 : 사용자가 요청했을 때 - 잘못된 요청, 요청한 리소스 없을 때, ...
500번대 : 서버 내부 문제, 오류, ...
